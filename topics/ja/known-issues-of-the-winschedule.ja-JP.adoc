= WinSchedule の既知の問題

このトピックでは、開発サイクル中に確認された WinSchedule コントロールの既知の問題を説明します。

== 予定のドラッグ

宛先へのパスにコントロールのスクロールが含まれる場合の予定または休日のドラッグでは、Drag アクションを MoveAppointment アクションに置き換える必要があります。これは、ドラッグ操作のグラス再生をさせないコントロール動作のために必要です。
