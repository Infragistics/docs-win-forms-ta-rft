= Supported Data Verification Points

You can use data verification points to test control data in a tested application. When you create a verification point, a snapshot of the verified data is stored. Then, during the replay session of the verification point, the data in the control is compared to the saved snapshot to see if any changes occurred. This way, you can identify potential errors in the application.

When using data verification points, you can use both literal values and data-driven values retrieved from the test data pool.

.Note
[NOTE]
====
For more information on working with verification points through IBM Rational Functional Tester, see the IBM Rational Software Development Platform Help (User Guide > Recording > Verification Points) installed with Rational Functional Tester.
====
